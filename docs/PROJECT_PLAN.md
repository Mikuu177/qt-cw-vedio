## Evaluation Methods Details

### Iteration 1: Heuristic Evaluation
- **Why**: Identify usability issues early in development
- **Participants**: 3-5 evaluators (peers or UX experts)
- **Process**:
  - Evaluators inspect UI against Nielsen's 10 heuristics
  - Document violations with severity ratings
  - Consolidate findings
- **Outcome**: List of usability issues to fix

### Iteration 2: Cognitive Walkthrough
- **Why**: Assess learnability of editing workflow
- **Participants**: 3-5 target users
- **Process**:
  - Define specific tasks (e.g., "trim a video clip")
  - Users perform tasks while thinking aloud
  - Observer documents confusion points
  - Analyze task completion success
- **Outcome**: Workflow improvements and UI clarifications

### Iteration 3: SUS Questionnaire
- **Why**: Quantify overall usability
- **Participants**: 5-10 users
- **Process**:
  - Users complete tasks with the system
  - Fill out 10-question SUS survey
  - Calculate SUS score (0-100)
  - Compare against industry benchmarks
- **Outcome**: Quantitative usability score and statistical analysis

## UI/UX Theory Application

### Principles to Apply
1. **Fitts's Law**: Larger, closer targets for frequent actions (play/pause)
2. **Hick's Law**: Minimize choices in menus, group related functions
3. **Gestalt Principles**: Visual grouping of related controls
4. **Consistency**: Follow platform conventions (Qt style)
5. **Feedback**: Visual/audio confirmation of user actions
6. **Error Prevention**: Confirm destructive actions, undo support
7. **Accessibility**: WCAG 2.1 guidelines

### Design Patterns
- Timeline scrubbing
- Drag-and-drop editing
- Context menus for quick actions
- Keyboard shortcuts for power users
- Progressive disclosure for advanced features

## Risk Management

### Technical Risks
- **FFmpeg integration complexity**: Allocate extra time, use libraries
- **Performance issues**: Early testing, optimization sprints
- **Cross-platform compatibility**: Focus on one platform primarily

### Schedule Risks
- **Evaluation recruitment delays**: Start early, have backup participants
- **Feature creep**: Stick to defined scope, prioritize ruthlessly
- **Last-minute bugs**: Buffer time in week 4

### Quality Risks
- **Insufficient evaluation data**: Over-recruit participants
- **Weak theory application**: Regular theory review sessions
- **Poor video quality**: Practice video creation early
